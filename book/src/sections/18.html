¶ Вдруг изменились приоритеты

А теперь допустим, в разработке изменились приоритеты, и вам нужно кроме предыдщих требований написать журнал игры.

В ТДД с внезапными изменениями требований проблем нет. Цикл разработки короткий, поэтому больших изменений он вызвать не может. Из-за этого у вас всегда есть рабочая версия проекта, которую можно дорабатывать как угодно.

В нашем примере, нам нужно изменить вектор разработки и начать пилить историю ходов.

Начнём с метода получения истории getMoveHistory.

test/game.js
it('Game saves user\'s move in history', () => { 
  const x = 1, y = 1

  game.acceptUserMove(x, y)
  const history = game.getMoveHistory()

  expect(history).to.deep.equal([{turn: 'user', x, y}])
})
